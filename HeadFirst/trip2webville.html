What solution are we using to the problem presented in the chapter of finding yourself a hosting company?
Our class is using github to host our websites currently, though it wasn't always that way.
What are the advantages and disadvantages of our solution?
github wasn't designed to be a go-to web hosting server, meaning we don't have full control over domain names and goodies are limited. On the other hand though, uploading files is simple, reliable, and free!
What alternatives could we have used, and which alternative did we use earlier in the year?
We could have used SiteGround, DreamHost, GoDaddy, or other hosting companies, however being free is probably the most important factor for our class given that we aren't master professional web designers at the moment. Earlier in the year we used the students.gctaa.net server, and I forget why we switched, but I will say personally github is a lot more conveinent than the student server ever was.
A few of our enterprising classmates have taken the initiative to register their own domain name. See if you can find these classmates and ask them how they did it.
I hadn't read this part at the beginning of the week, but I was in the room listening to him when he was shopping around for one. He was on different websites that sold domain names and read through there selections, trying to find one suited to his desires and budget.
What is the process involved?
I also did some of my own research for this, in order to get a domain name, here's one way to go about it:
  1. Decide a domain name you want for your site.
  2. Shop around multiple sites to find the best deal for your domain name or a similar domain name. Often the web hosting servers will also offer domain names. Additionally, domain availability checker tools exist which can aid in the search.
  3. Complete the transaction (usually domains are purchasable through the site they are found on).
  4. Verify the domain name (you may get an email after your purchase asking you to do this).
Where can you go to register a domain name?
As previously mentioned, the web hosting companies often allow you to buy domain names from them. It is also possible to buy a domain name from a person, provided you negotiate a fair price.
Compare the workflow presented in the book (which is a bit old at this point) with the one we use. How are they different?
The book describes having to first connect to the server using an ftp command and putting in a username & password. github requires your ssh key, but edits can be made while not connected to the server and can be pushed all at once later.
How are they similar?
When we used the student server, the workflow was essentially identical besides using ssh instead of ftp. github does have similarities as well, such as having to put in a password (Even if it isn't a custom password and rather just our public/private keys) and that connections have to be made with the git command to update the server (even if updating the server doesn't require working while connected to the web server).
